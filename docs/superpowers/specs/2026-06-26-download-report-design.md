# Скачать отчёт со страницы «Статистика»

Дата: 2026-06-26

## Цель

Добавить на страницу «Статистика» (`/stats`) кнопку **«Скачать отчёт»** рядом с
«Свернуть все». По клику открывается модалка, где пользователь выбирает период и
формат, и скачивает красиво оформленный отчёт с цифрами за период и графиками.

## Требования

- Кнопка `Скачать отчёт` в правом верхнем углу страницы Статистика, **перед**
  кнопкой «Свернуть все». Иконка `Download`, `variant="outline"`, `size="sm"`.
- Модалка на базе `ModalWrapper`:
  - **Dropdown «Период»** — выбор одного периода (как на странице Графики).
  - **Выбор формата** — карточки-плитки с иконкой и подписью, **одиночный выбор**:
    - **PDF** — «Цифры + графики»
    - **PNG** — «Снимок отчёта»
    - **CSV** — подпись **«только цифры»**
  - Футер: **Отмена** (outline) + **Скачать отчёт** (primary, спиннер при генерации).
- Охват: цифры — за выбранный период; графики — кумулятивно от самого старого
  периода до выбранного включительно (идентично логике `ChartsSection`).
- Графики в отчёте: CFD + остаток критичных + остаток некритичных (все три со
  страницы Графики).
- Комментарий периода (если есть у зафиксированной статистики) включается в отчёт
  (блок в PDF/PNG, отдельная строка в CSV).
- Имя файла: `stability-report_ДД.ММ.ГГГГ.{pdf|png|csv}`.

## Архитектура

### Источники данных (переиспользуем существующее)
- Цифры периода: `statistics ?? calculateDynamicMetrics(period, periods, tasks, periodStatistics)`
  — та же логика, что в `StatsPeriodCard` (`metrics = statistics ?? dynamicMetrics`).
- Данные графиков: `calculateChartData(periods, tasks, periodStatistics)`, срез
  `slice(0, selectedSortedIndex + 1)` — как в `ChartsSection`.

### Компоненты
- `DownloadReportButton` — кнопка + состояние открытия модалки. Встраивается в
  заголовок `src/app/stats/page.tsx`.
- `DownloadReportModal` — выбор периода + формата, кнопка скачивания, состояние
  генерации. На базе `ModalWrapper`.
- `FormatCard` — плитка выбора формата (иконка + заголовок + подпись), радио-логика.
- `ReportDocument` — оффскрин-вёрстка отчёта (шапка, блоки цифр, графики).
  Рендерится за пределами вьюпорта с фиксированной шириной для надёжного снимка.
- `ReportCharts` (CFD + 2 backlog) — отдельные графики для отчёта с фиксированным
  размером и `isAnimationActive={false}`, чтобы SVG был полностью отрисован до снимка.

### Генерация (динамические импорты)
- `lib/report/generate-png.ts` — `html-to-image` → PNG.
- `lib/report/generate-pdf.ts` — `html-to-image` → постранично A4 через `jsPDF`.
- `lib/report/generate-csv.ts` — строки из метрик, UTF-8 BOM + разделитель `;`
  (дружелюбно к Excel-RU). Без рендеринга.
- `lib/report/download.ts` — общий хелпер скачивания Blob/dataURL.

### Технологии
- `html-to-image` вместо `html2canvas`: Tailwind v4 использует `oklch()`-цвета,
  которые `html2canvas` не поддерживает (падает). `html-to-image` рендерит через
  SVG `foreignObject`, цвета считает браузер.
- `jsPDF` для обёртки снимка в PDF (A4, многостраничность по высоте).
- Обе библиотеки импортируются динамически внутри обработчика — не попадают в
  основной бандл, нет проблем с SSR.

## Поток

1. Пользователь жмёт «Скачать отчёт» → открывается модалка (период по умолчанию —
   самый свежий).
2. Выбирает период и формат, жмёт «Скачать отчёт».
3. Считаются метрики и данные графиков для периода.
4. CSV: формируется и скачивается напрямую.
   PDF/PNG: `ReportDocument` монтируется оффскрин, ждём кадр (rAF + tick) для
   отрисовки графиков, делаем снимок, скачиваем (PNG) или собираем PDF, размонтируем.
5. Модалка закрывается.

## Вне рамок (YAGNI)

- Диапазон из нескольких периодов (выбран только один период).
- Серверная генерация / сохранение отчётов в БД.
- Настройка состава отчёта пользователем.
